# 销售情况仪表盘

这是一个基于 Excel 的销售分析项目，核心目标是把分散的月度成交明细整合为可复用的数据分析模型，并最终输出可交互的销售仪表盘。

项目的主线非常明确：

`Power Query 做 ETL -> 数据模型沉淀到中间表/透视表 -> 切片器联动分析 -> GETPIVOTDATA 取数搭建仪表盘`

## 项目结构

- `销售情况仪表盘.xlsx`：主分析文件，包含 ETL、透视分析和仪表盘展示
- `数据处理/6月_每日销售成交汇总数据.xlsx`：6 月销售成交明细
- `数据处理/7月_每日销售成交汇总数据.xlsx`：7 月销售成交明细
- `数据处理/历史数据.xlsx`：历史成交沉淀数据
- `数据处理/销售人员表-截止8月1日.xlsx`：销售人员基础信息

## 使用技术

- Power Query
  用于完成数据抽取、清洗、转换与整合，是整个项目的 ETL 核心。
- Excel 数据透视表
  用于对成交金额、成交客户数、客单价等指标按月份、区域、省份、产品类型等维度聚合分析。
- 切片器
  使用月份、区域、产品类型等切片器实现多透视表联动筛选，提高交互性。
- GETPIVOTDATA 函数
  用于从中间透视表稳定取数，把分析结果映射到仪表盘固定展示区域。
- 条件格式与图表
  用于突出金额、占比、环比增速和趋势变化，增强可视化表达。
- Git 与 GitHub
  用于版本管理、项目归档和成果展示。

## 仪表盘制作思路

### 1. 用 Power Query 做 ETL

先把不同月份的销售明细、历史数据和销售人员信息接入 Excel。

在 Power Query 中完成以下处理：

- 统一字段口径
- 规范日期与月份字段
- 补充区域、省份、产品类型、期数等分析维度
- 整合历史数据与新增数据
- 输出适合透视分析的结构化明细表

这一层的目标不是展示，而是保证后续所有透视分析都建立在统一、干净、可复用的数据底座上。

### 2. 用透视表搭建分析中间层

项目没有直接拿原始明细做仪表盘，而是先建立一层“中间表/中间透视表”。

这一层承担了三个作用：

- 聚合核心指标，如成交金额、成交客户数、客单价
- 按月份、区域、省份、产品类型、期数等维度拆分结果
- 为仪表盘提供稳定的取数来源

从工作簿结构看，项目中存在多张透视表，用于支持主指标、区域明细、产品类型分析、区域趋势和环比计算。

### 3. 用切片器实现交互筛选

仪表盘使用了月份、区域、产品类型等切片器。

切片器不是简单装饰，而是把多个透视分析对象串联起来，让用户可以从不同维度快速观察结果变化。

这说明项目在设计时已经考虑了“业务人员如何使用”，而不是只停留在静态展示。

### 4. 解决环比指标与月份切片器冲突

这是整个项目里最关键、也最能体现分析建模思路的一点。

如果环比增长率直接依赖月份切片器联动的透视表，那么当月份被切成单月时，差异百分比会失去对比基准，环比结果就会消失。

为了解决这个问题，项目采用了两层处理：

- 将计算环比增长率的数据透视表切断与月份切片器的直接关联
- 额外新增“月份筛选字段”，让 `GETPIVOTDATA` 在引用透视表时仍然可以根据月份定位结果

这本质上是在区分两件事：

- 展示筛选逻辑
- 计算逻辑

也就是说，用户看到的月份切换，不再直接破坏环比的计算基础。这是很成熟的数据分析建模思路。

### 5. 用 GETPIVOTDATA 固化仪表盘取数

仪表盘页不是直接展示透视表，而是通过 `GETPIVOTDATA` 从中间透视表抽取指标。

这样做有三个好处：

- 仪表盘布局可以完全独立于透视表位置
- 指标引用更稳定，不容易因为拖动透视表而失效
- 可以把不同透视表中的结果组合到同一个展示界面

从工作簿结构看，仪表盘中不仅引用了核心 KPI，也引用了区域、省份、产品类型等分层分析结果。

## 数据分析思维提炼

这个项目体现出的分析思维，不只是“会做图表”，而是完整的数据分析闭环：

- 先治理数据，再分析数据
  先用 Power Query 统一口径，而不是直接对原始明细做展示。
- 先搭分析层，再搭展示层
  通过中间透视表把指标体系稳定下来，再用仪表盘做呈现。
- 把计算逻辑和展示逻辑分开
  这是环比问题能被正确解决的根本原因。
- 从总览到拆解逐层分析
  先看总成交，再看区域，再看省份，再看产品类型和期数。
- 兼顾结果指标和过程结构
  不只看成交金额，还看客户数、客单价、占比和环比，避免单指标误判。

## 当前可提炼的指标体系

项目中已经沉淀出一套较标准的销售分析指标：

- 成交金额
- 成交客户数
- 客单价
- 成交金额占比
- 成交客户数占比
- 金额环比增长率
- 客户数环比增长率
- 客单价环比增长率

## 项目亮点

- 不是简单 Excel 报表，而是带 ETL 流程的数据分析作品
- 通过透视表中间层提高了模型复用性
- 正确处理了月份切片器与环比计算的冲突
- 通过 `GETPIVOTDATA` 提升了仪表盘展示层稳定性
- 已经具备标准 BI 仪表盘的基本方法论雏形
